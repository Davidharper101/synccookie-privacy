Privacy Policy for Syncookie Extension

Effective Date: June 3, 2026

1. Overview

The Syncookie browser extension (“the Extension”) is designed for personal cookie management. It automatically captures cookies from the active browser tab and securely transmits them to a server owned and controlled by you, the user. This policy explains what data is collected, how it is handled, and your rights.

2. Data Collection

The Extension reads only the cookies associated with the website displayed in your active browser tab, including any embedded frames (iframes). It does not collect:

  • Browsing history
  • Passwords
  • Form data
  • Personally identifiable information (PII)
  • Screenshots, keystrokes, or any other page content

Each cookie object contains standard fields as provided by the browser’s API (name, value, domain, path, secure flag, httpOnly flag, sameSite attribute, expiration date, etc.). No additional data is extracted or inferred.

3. Data Transmission

All cookie data is encrypted locally in your browser using AES‑256‑GCM before any network request is made. The encryption key is fetched from your personal server at the moment of transmission and is never stored by the Extension.

The encrypted payload is sent exclusively to the server URL you have configured (by default https://synccookie.com, but you may change it to your own VPS). Transmission always occurs over HTTPS, ensuring the connection itself is also encrypted.

4. Data Storage and Server

The destination server is your own infrastructure. The Extension developer does not operate any data‑receiving servers. You are solely responsible for the security and configuration of that server.

Once the encrypted data arrives at your server, it is decrypted using the same AES key and stored according to your server’s settings (for example, in a local JSON file). No third‑party service, cloud provider, or external database is involved.

5. Data Use

The Extension is intended for personal use only. Typical use cases include:

  • Backing up cookies for later import
  • Transferring cookies between devices
  • Debugging or analysing your own web sessions

The Extension does not use your data for any commercial purpose, advertising, tracking, or analytics. The developer does not have access to your data (unless you explicitly share your server credentials or files).

6. Data Sharing

Cookie data is never shared, sold, or disclosed to any third party. All data remains entirely within the chain between your browser and your personal server.

7. Security

  • Encryption: AES‑256‑GCM ensures cookie values cannot be read in transit even if HTTPS were compromised.
  • Key Handling: The decryption key exists only on your server and is fetched fresh for each transmission.
  • Transmission: All network requests use HTTPS (TLS 1.2/1.3).
  • Local Storage: The Extension stores only the configured server URL and a cooldown timestamp; it does not store cookies locally.

8. User Control

  • Configuration: You can change the target server URL by editing the Extension’s source code (SERVER_URL in background.js).
  • Data Deletion: Since all data resides on your own server, you can delete it at any time.
  • Disabling/Uninstalling: Removing the Extension leaves no residual data in the browser.
